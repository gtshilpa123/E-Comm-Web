mongoose is mongodb Object Data Modeling(ODM) for Node.js
ORM -> Object Relational Mappers, which are used with relational databases.
Mongoose give validation at the database level.
Advantages of Mongoose are schema definition, middleware support, and simplified querying.

npm i mongoose

Everything in mongoose starts with schema.
Id 's are internally managed by mongoose
Schema types in mongoose includes, String, Number, ObjectId

Based on schema, models are created. Model is mapped to a collection. Every instance of model is called document. On document, we can perform CRUD operations.

Express validator will do validation at the API level.
At the data level, validations can be done using mongoose schema.

1. One to One -> one document inside a collection maps to another document in another collection.

Middlewares normally intercept the request and response where you can modify the request and responses as per need.

Middlewares in mongoose intercept the database operations. Before you perform database operation if you want to do something and after that database operation is performed and you want to do something. For these 2 interception points, 2 hooks named pre and post hooks functions are used. 
These hooks are functions which are passed control during execution of asynchronous functions.

Database Best Practices: Schema design, Indexing, Error handling, Validations.

Mongoose schema types specifies data types and applying required and custom validators as needed. It ensure data integrity and consistency by enforcing validation rules.
